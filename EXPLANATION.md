# Объяснение параметров PPO LAG в TTCT

## Параметры обучения

### 1. `--use_pretrained_encoders` (или `--use-pretrained-encoders`)
**Что это:** Булев флаг, определяющий, нужно ли загружать предобученные веса энкодеров из модели TTCT.

**Как работает:**
- Если `True`: Загружает веса из файла, указанного в `--TL-loadpath`, в модель `EncodeModel` (если используется fine-tuning) и в модель `TLmodel` (основная модель для предсказания стоимости).
- Если `False`: Модели инициализируются случайными весами.

**Когда использовать:**
- `True`: Когда у вас уже есть обученная модель TTCT и вы хотите использовать её для обучения политики.
- `False`: Когда вы начинаете обучение с нуля или хотите обучить модель TTCT заново.

**Код:** Строки 165-166, 188-189 в `ppo_lag.py`

---

### 2. `--use-predict-cost`
**Что это:** Булев флаг, определяющий, использовать ли модель TTCT для предсказания стоимости (cost) вместо ground-truth стоимости из окружения.

**Как работает:**
- Если `True`: Использует модель TTCT (`TLmodel`) для предсказания стоимости траектории на основе текстовых ограничений и истории наблюдений/действий.
- Если `False`: Использует реальную стоимость (`true_cost`) из окружения.

**Когда использовать:**
- `True`: Когда вы хотите использовать обученную модель TTCT для оценки стоимости траекторий на основе текстовых ограничений.
- `False`: Когда вы хотите использовать стандартную ground-truth функцию стоимости (для сравнения или базовой линии).

**Код:** 
- Строка 142: `use_predict_cost=args.use_predict_cost`
- Строка 417: Передаётся в `buffer.finish_path()` как `TL_condition`
- Строка 435: Используется для выбора между предсказанной и реальной стоимостью

---

### 3. `--use-credit-assignment` (или `--use_credit_assignment`)
**Что это:** Булев флаг, определяющий, использовать ли компонент credit assignment модели TTCT для распределения стоимости по шагам траектории.

**Как работает:**
- Если `True`: Использует `cost_assignment_layer` модели TTCT для распределения общей стоимости эпизода по отдельным шагам траектории.
- Если `False`: Использует только `episodic_cost_layer` для предсказания общей стоимости эпизода без распределения по шагам.

**Когда использовать:**
- `True`: Когда вы хотите использовать полную функциональность TTCT, включая распределение стоимости по шагам (credit assignment).
- `False`: Когда вы хотите использовать только предсказание общей стоимости эпизода без детального распределения.

**Код:**
- Строка 417: Передаётся в `buffer.finish_path()` как `use_cost_prediction`
- В `buffer.py` строка 133: Вызывается `TLmodel.get_cost()` с параметром `use_cost_prediction`

---

### 4. `--TL-loadpath` (или `--TL_loadpath`)
**Что это:** Путь к файлу с сохранёнными весами обученной модели **TTCT (Trajectory-level Textual Constraints Translator)**.

**TTCT модель - что это:**
TTCT - это нейросетевая модель, которая:
- Переводит текстовые ограничения (например, "не наступай на красные клетки") в числовые оценки стоимости траекторий
- Кодирует траектории (последовательности наблюдений и действий) в векторные представления
- Предсказывает стоимость траекторий на основе текстовых ограничений
- Может распределять стоимость по отдельным шагам траектории (credit assignment)

**Как работает:**
- Указывает путь к файлу `.pth` или `.pt`, содержащему `state_dict` модели TTCT (веса всех слоёв)
- Используется функцией `load_from_save()` для загрузки весов в модель
- Модель TTCT должна быть предварительно обучена командой `python train.py` (см. README)

**Когда использовать:**
- **Обязательно**, если `--use_pretrained_encoders=True` - для загрузки весов в модель энкодера
- **Обязательно**, если `--use-predict-cost=True` - для использования TTCT для предсказания стоимости
- Должен указывать на **существующий файл** с обученной моделью TTCT

**Примеры путей:**
```bash
--TL-loadpath=/path/to/trained_ttct_model.pth
--TL-loadpath=./models/ttct_checkpoint_100.pth
--TL-loadpath=/home/user/ttct_models/best_model.pt
```

**Важно:**
- Файл должен быть сохранён после обучения TTCT (команда `python train.py`)
- Если файл не существует, будет выдана ошибка с понятным сообщением
- Если указать неполный путь (например, `/yo`), обучение не запустится

**Код:**
- Строка 152: `TL_loadpath=args.TL_loadpath`
- Строки 171, 194: Используется в `load_from_save()` для загрузки весов

## Режимы обучения (из README)

### 1. Полный TTCT (с credit assignment):
```bash
python ppo_lag.py --use-predict-cost --use-credit-assignment --lagrangian-multiplier-init=0.1 --TL-loadpath=/your/TTCT/model/path
```

### 2. TTCT без credit assignment:
```bash
python ppo_lag.py --use-predict-cost --lagrangian-multiplier-init=0.1 --TL-loadpath=/your/TTCT/model/path
```

### 3. Стандартный режим (ground-truth cost):
```bash
python ppo_lag.py --lagrangian-multiplier-init=0.1 --TL-loadpath=/your/TTCT/model/path
```

## Обнаруженные проблемы и исправления

### Проблема 1: NameError при использовании EncodeModel
**Ошибка:** В коде вызывается `EncodeModel.train()` и другие методы `EncodeModel`, но переменная `EncodeModel` определена только если `args.is_finetune=True`.

**Исправлено:** Добавлены проверки `if args.is_finetune:` перед всеми использованиями `EncodeModel`:
- Строка 212: `EncodeModel.train()`
- Строка 306: `logger.setup_torch_saver1(EncodeModel)`
- Строка 612: `Encode_scheduler.step()`

### Проблема 2: Парсинг булевых аргументов
**Ошибка:** Параметры `--use_pretrained_encoders`, `--use-lora`, и `--is-finetune` были определены как `type: bool` в `config.py`, что неправильно обрабатывает строковые значения. Когда вы передаёте `--use_pretrained_encoders false`, argparse получает строку "false", и `bool("false")` в Python возвращает `True` (любая непустая строка = True).

**Исправлено:** Изменён парсинг на `lambda x: bool(strtobool(x))` для правильной обработки строковых значений "true"/"false", "1"/"0", "yes"/"no".

**Теперь работает правильно:**
- `--use_pretrained_encoders false` → `False`
- `--use_pretrained_encoders true` → `True`
- `--use_pretrained_encoders 0` → `False`
- `--use_pretrained_encoders 1` → `True`

### Проблема 3: Отсутствие проверки существования файла модели
**Ошибка:** Функция `load_from_save()` пыталась открыть файл без проверки его существования, что приводило к неинформативным ошибкам.

**Исправлено:** Добавлена проверка существования файла с понятным сообщением об ошибке.

### Проблема 4: Предупреждение при использовании случайных весов
**Добавлено:** Предупреждение, когда используется `--use-predict-cost` без загруженной модели (случайные веса могут не работать правильно).

## Дополнительные параметры

- `--is-finetune`: Определяет, нужно ли fine-tune модель энкодера во время обучения политики.
- `--use-lora`: Использовать ли LoRA (Low-Rank Adaptation) для fine-tuning модели.
- `--language-model`: Какую языковую модель использовать ('TLmodel' или 'Bert').
