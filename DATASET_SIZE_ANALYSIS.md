# Анализ размера датасета согласно статье TTCT

## Информация из статьи

### Количество текстовых ограничений:
Из раздела **6.1 Setup** статьи:
> "we designed **over 200 trajectory-level textual constraints** which can be grouped into 4 categories"

**Вывод:** Более 200 различных текстовых ограничений.

### Разбиение датасета:
Из раздела **A.1 Dataset** статьи:
> "Then we randomly split the (trajectory, textual constraint) pairs into **80% training and 20% test sets**"

**Вывод:** Разбиение 80% train / 20% test.

### Процесс создания датасета:
Из раздела **A.1 Dataset** статьи:
> "we first explore the environment using a random policy, **collecting a large amount of offline trajectory data**"

**Вывод:** Собирается большое количество офлайн траекторий.

## Оценка размера датасета

### Минимальные требования:

**Исходя из batch_size=194 (по умолчанию в коде):**
- Для одного батча нужно минимум 194 пары (trajectory, constraint)
- Для одной эпохи при batch_size=194 нужно: `194 * количество_батчей`
- Для 32 эпох нужно достаточно данных для обучения

**Оценка:**
- **Минимум:** ~1000-2000 пар (trajectory, constraint) для базового обучения
- **Рекомендуется:** ~5000-10000 пар для стабильного обучения
- **Для воспроизведения результатов статьи:** вероятно, 10000+ пар

### Расчет на основе ограничений:

Если у нас **200+ текстовых ограничений** и каждая траектория может нарушать несколько ограничений:

**Сценарий 1: Одна траектория = одно ограничение**
- Нужно минимум 200 траекторий (по одной на каждое ограничение)
- Но это слишком мало для обучения

**Сценарий 2: Каждая траектория нарушает 2-3 ограничения**
- 200 ограничений × 10-20 траекторий на ограничение = **2000-4000 траекторий**
- При `num_pairs_per_trajectory=2`: **4000-8000 пар**

**Сценарий 3: Большой датасет (как в статье)**
- Вероятно, **10000+ траекторий**
- При `num_pairs_per_trajectory=2`: **20000+ пар**

## Рекомендации по размеру датасета

### Для тестирования (быстрый запуск):
```bash
python generate_dataset_from_paper.py \
    --num_trajectories 500 \
    --num_pairs_per_trajectory 2
```
**Результат:** ~1000 пар (trajectory, constraint)
- Train: ~800 пар
- Test: ~200 пар

### Для реального обучения (рекомендуется):
```bash
python generate_dataset_from_paper.py \
    --num_trajectories 2000 \
    --num_pairs_per_trajectory 2
```
**Результат:** ~4000 пар (trajectory, constraint)
- Train: ~3200 пар
- Test: ~800 пар

### Для воспроизведения результатов статьи:
```bash
python generate_dataset_from_paper.py \
    --num_trajectories 5000 \
    --num_pairs_per_trajectory 2
```
**Результат:** ~10000 пар (trajectory, constraint)
- Train: ~8000 пар
- Test: ~2000 пар

## Размер файла датасета

**Оценка размера одной пары:**
- `obs`: [length, 7, 7, 3] × 4 bytes = ~length × 588 bytes
- `act`: [length] × 4 bytes = ~length × 4 bytes
- При средней длине 120 шагов: ~70 KB на пару

**Размер датасета:**
- 1000 пар: ~70 MB
- 4000 пар: ~280 MB
- 10000 пар: ~700 MB

## Выводы

### Минимальный размер для работы:
- **~1000 пар** (500 траекторий × 2 пары) - для тестирования

### Рекомендуемый размер:
- **~4000-5000 пар** (2000-2500 траекторий × 2 пары) - для реального обучения

### Для воспроизведения статьи:
- **~10000+ пар** (5000+ траекторий × 2 пары) - вероятный размер в статье

### Важно:
- В статье не указано точное число траекторий
- Указано "large amount" - значит достаточно много
- Разбиение 80/20 означает, что нужен достаточно большой датасет для стабильного обучения
