# Обновление: Проверка нарушений для CLIP-подобного обучения

## Что изменилось

### Проблема
Ранее скрипт `generate_dataset_from_paper.py` случайно выбирал ограничения для траекторий, **не проверяя**, действительно ли траектория нарушает каждое ограничение. Это неправильно для CLIP-подобного контрастного обучения.

### Решение
Теперь для **каждой траектории** проверяются **ВСЕ возможные ограничения**, и в датасет включаются:
- ✅ **Положительные пары**: траектория нарушает ограничение (`mask[i,j] = 1.0`)
- ✅ **Отрицательные пары**: траектория НЕ нарушает ограничение (`mask[i,j] = 0.0`)

## Новые функции

### 1. `analyze_trajectory_stats(trajectory)`
Анализирует траекторию и возвращает статистику:
- Количество шагов по лаве, воде, траве
- Последовательность объектов
- Длину траектории

### 2. `check_violation(trajectory, constraint)`
Проверяет, нарушает ли траектория конкретное ограничение:
- **Quantitative**: подсчитывает шаги по объектам
- **Sequential**: проверяет последовательности действий
- **Mathematical**: вычисляет HP и проверяет смерть
- **Relational**: проверяет расстояние до опасностей

### 3. `generate_all_possible_constraints()`
Генерирует все возможные ограничения (~50+ ограничений):
- Quantitative: 16 ограничений (4 значения × 4 типа)
- Sequential: 5 ограничений
- Mathematical: 6 ограничений (3 HP × 2 шаблона)
- Relational: 3 ограничения

### 4. `create_dataset_pairs()` (обновлена)
Теперь:
- Проверяет **ВСЕ** ограничения для каждой траектории
- Включает в датасет **ВСЕ** пары (нарушенные и не нарушенные)
- Выводит статистику нарушений

## Изменения в использовании

### Раньше:
```bash
python generate_dataset_from_paper.py \
    --num_trajectories 500 \
    --num_pairs_per_trajectory 2
```
Результат: ~1000 пар (500 × 2)

### Теперь:
```bash
python generate_dataset_from_paper.py \
    --num_trajectories 500
```
Результат: ~25000+ пар (500 траекторий × ~50 ограничений)

**Важно:** Теперь для каждой траектории проверяются ВСЕ ограничения автоматически!

## Размер датасета

### Пример расчета:
- **500 траекторий** × **~50 ограничений** = **~25000 пар**
- После разбиения 80/20:
  - Train: ~20000 пар
  - Test: ~5000 пар

### Статистика нарушений:
Скрипт выводит статистику:
```
Итого пар создано: 25000
  Нарушено: 5000 (20.0%)
  Не нарушено: 20000 (80.0%)
```

## Преимущества

1. ✅ **Правильное контрастное обучение**: есть и положительные, и отрицательные пары
2. ✅ **Больше данных**: каждая траектория создает ~50 пар вместо 2
3. ✅ **Автоматическая проверка**: не нужно вручную выбирать ограничения
4. ✅ **Соответствие статье**: как описано в Section 4.1 (CLIP-подобное обучение)

## Примечания

- Генерация датасета теперь занимает больше времени (проверка всех ограничений)
- Размер датасета значительно увеличивается
- Для тестирования можно использовать меньше траекторий: `--num_trajectories 100`
